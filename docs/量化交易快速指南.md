# 🧪 量化模拟与历史回放快速指南

这份指南描述的是**当前代码里的量化验证能力**，重点是：

- `量化模拟`
- `历史回放`
- 共享 `量化候选池`

这和旧文档里“MiniQMT 自动实盘交易”不是一回事。  
当前主程序内置的是**模拟账户验证流程**。

---

## 当前量化主线

### 1. 我的关注

先在工作台的 `我的关注` 里维护你真正关心的股票。

来源可以是：

- 手工输入
- 发现股票页的选股结果
- 研究情报页中明确产出的股票

### 2. 量化候选池

从 `我的关注` 里单选或多选股票，加入 **量化候选池**。

量化候选池是：

- `量化模拟` 的输入池
- `历史回放` 的输入池

### 3. 量化模拟

围绕量化候选池做实时或定时模拟：

- 定时任务配置
- 实时信号
- 自动执行模拟交易
- 账户结果
- 持仓和权益

### 4. 历史回放

围绕同一个量化候选池做历史验证：

- 区间回放
- 连续模拟
- 回放任务
- 回放结果
- 成交和权益曲线

---

## 一、量化模拟

### 1. 页面结构

当前 `量化模拟` 页面主要分成：

- 左侧：定时任务配置 + 运行状态
- 右侧：账户结果 + 量化候选池 + 执行中心

### 2. 重点配置

- 资金池
- 分析粒度：`30m / 1d / 1d+30m`
- 策略模式：`自动 / 激进 / 中性 / 稳健`
- 自动执行模拟交易

### 3. 自动执行怎么理解

自动执行是**模拟账户自动成交**，不是实盘下单。

它会遵守当前规则：

- A 股最小交易单位是 `100` 股
- 没有有效价格不成交
- 没有可卖持仓不卖出

如果自动执行被跳过，页面会直接告诉你原因，例如：

- `自动执行跳过：建议仓位不足买入一手`
- `自动执行跳过：当前无可卖持仓`
- `自动执行跳过：缺少有效最新价`

### 4. 为什么会“开了自动执行却没有成交”

最常见不是任务坏了，而是：

- 当前轮次大多是 `HOLD`
- 账户现金不足买一手
- 当前策略建议仓位太小

所以要把“自动执行”理解成：

> 自动执行满足交易规则的模拟信号，而不是无条件强制下单。

## 二、历史回放

### 1. 页面结构

当前 `历史回放` 页面是：

- 左侧：回放配置、运行概况、任务列表、只读量化候选池
- 右侧：任务详情、回放结果、持仓、权益曲线、成交明细、信号执行记录

### 2. 回放模式

支持：

- 历史区间回放
- 从过去接续到当前模拟

### 3. 回放结果里能看什么

- 回放总览
- 最终现金、最终持仓市值、最终总权益
- 收益率、回撤、胜率、交易笔数
- 当前持仓结果
- 资金曲线
- 成交明细
- 信号执行记录

### 4. 为什么有时看起来“没有执行信号”

不是一定没数据，常见原因有：

- 最近时间点大多是 `HOLD`
- 页面默认先看到的是中性信号
- 回放信号有，但成交数量为 `0`

如果需要判断是否真的有执行信号，建议重点看：

- `成交明细`
- `交易笔数`
- `信号执行记录` 里的 `BUY / SELL`

---

## 三、策略说明怎么看

现在量化模拟和历史回放里的策略详情，已经调整成 3 层：

### 第一层：人话结论

- 当前判断
- 操作建议
- 核心原因
- 风险与观察点

### 第二层：策略解释

- 市场状态
- 基本面质量
- 当前风格
- 时间框架
- 建议仓位

### 第三层：量化证据

可展开查看：

- 技术证据
- 环境证据
- 双轨裁决
- 执行提示

这样既保留量化审计证据，也尽量让用户能先看懂结论。

## 四、推荐操作路径

### 路径 A：实时量化模拟

1. 在工作台把股票加入 `我的关注`
2. 再从 `我的关注` 推入量化候选池
3. 打开 `量化模拟`
4. 配置资金池、粒度、策略模式和自动执行
5. 启动定时任务
6. 看账户结果、持仓和信号

### 路径 B：历史验证

1. 先准备量化候选池
2. 打开 `历史回放`
3. 选择回放区间和模式
4. 启动回放
5. 看结果、曲线、成交和信号
