# 🚀 快速开始

这份指南按照当前版本的真实主线来写：

**工作台 -> 我的关注 -> 发现股票 / 研究情报 -> 量化候选池 -> 量化模拟 / 历史回放**

---

## 1. 安装依赖

```bash
pip install -r requirements.txt
```

---

## 2. 配置 `.env`

复制模板：

```bash
copy .env.example .env
```

最低建议配置：

```env
DEEPSEEK_API_KEY=your_api_key
DEFAULT_MODEL_NAME=deepseek-chat
TDX_ENABLED=true
```

可选配置：

- `TUSHARE_TOKEN`
- 邮件 / Webhook 通知
- MiniQMT 相关接口

更多说明：

- [环境配置快速指南.md](环境配置快速指南.md)
- [TDX数据源配置指南.md](TDX数据源配置指南.md)

---

## 3. 启动系统

```bash
streamlit run app.py
```

打开：

[http://127.0.0.1:8501](http://127.0.0.1:8501)

---

## 4. 先看工作台

启动后默认进入 **工作台**。

这里最重要的是：

- `我的关注`
- 股票分析
- 右侧的下一步入口

如果你已经有想跟踪的股票：

1. 直接在 `我的关注` 输入股票代码
2. 刷新报价
3. 继续分析或推进到量化候选池

---

## 5. 从选股开始

如果你还没有目标股票：

1. 打开 `发现股票`
2. 在主力选股、低价擒牛、小市值、净利增长、低估值里运行筛选
3. 在结果页里把股票加入 `我的关注`
4. 回到工作台继续跟踪

---

## 6. 从研究开始

如果你想先看情报：

1. 打开 `研究情报`
2. 查看智策板块、智瞰龙虎、新闻流量、宏观分析、宏观周期
3. 如果模块里有明确股票输出，再加入 `我的关注`

---

## 7. 使用量化模拟

1. 先在工作台的 `我的关注` 里勾选股票
2. 批量加入 `量化候选池`
3. 打开 `量化模拟`
4. 配置：
   - 资金池
   - 分析粒度
   - 策略模式
   - 自动执行模拟交易
5. 启动定时任务或查看当前账户结果

说明：

- 自动执行是**模拟账户自动成交**
- 不是实盘下单
- 如果建议仓位不足买入 A 股一手，系统会明确提示跳过原因

---

## 8. 使用历史回放

1. 先确保量化候选池里已经有股票
2. 打开 `历史回放`
3. 选择回放模式、时间区间、粒度、策略模式
4. 启动回放
5. 查看回放任务、结果、持仓、资金曲线和成交记录

---

## 9. 推荐上手顺序

如果你是第一次使用，建议这样走：

1. 配好 `.env`
2. 打开工作台
3. 手工加 1-2 只股票到 `我的关注`
4. 体验一次单股分析
5. 去 `发现股票` 跑一次主力选股
6. 把几只股票加入 `我的关注`
7. 再把其中 2-3 只推进到量化候选池
8. 体验一次量化模拟和一次历史回放

---

## 10. 常见问题

### 为什么选股结果没有自动出现在我的关注？

当前设计是**手动加入**，不是自动沉淀。这样可以避免每次筛选都把大量候选股自动塞进主池。

### 为什么开启自动执行后，还是没有自动成交？

最常见原因是：

- 最近信号都是 `HOLD`
- 建议仓位不足买入 A 股一手
- 当前没有可卖持仓
- 缺少有效最新价

页面会直接给出 `自动执行提示`。

### 为什么量化模拟和历史回放都要求先有量化候选池？

因为它们现在共用同一个输入池，目的是让实时验证和历史验证围绕同一组股票展开。
